Setup

Connect Gemini safely, or keep using Demo Mode locally.

Trend2Short AI works without any backend. You can keep the product in Demo Mode with no API key, or switch to Live Mode by adding a Gemini API key to the local config file on your machine only.

1. Get A Gemini API Key

Create a Gemini API key from Google AI Studio. Keep that key private and never commit it to GitHub.

Official setup starts at ai.google.dev.

2. Place The Key In Vercel

Do not put the Gemini key in config.js.

The key belongs in Vercel only: 

Project Settings
Environment Variables
GEMINI_API_KEY

After adding the variable, redeploy the project. Keep config.js for non-secret frontend settings only.

Demo Mode

If GEMINI_API_KEY is missing in Vercel, the serverless function reports Demo Mode and the frontend uses the local browser-based generator as a fallback.

Live Mode

If GEMINI_API_KEY is present in Vercel, Trend2Short AI sends requests to /api/generate, and the serverless function calls Gemini 2.5 Flash and expects a JSON-only response for hooks, ideas, scripts, captions, hashtags, and CTAs.

Provider Architecture

Gemini is the first provider only. The service layer in js/services/ai.js is organized so future providers such as OpenAI, Claude, and OpenRouter can be added without rewriting the UI.

Rate Limit

The MVP enforces a local daily limit of 10 AI generations using LocalStorage. This keeps the front-end flow predictable while the product remains browser-only.
